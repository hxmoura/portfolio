   ### Real-time donation flow

<ul className="flex flex-col gap-4">
  <li className="animation-blur">• **User action:** The viewer visits the streamer’s page, enters their name and a message, and selects the donation amount. The system generates a Pix QR code for payment.</li>
  <li className="animation-blur">• **Confirmation:** As soon as the payment is completed, the payment gateway notifies our system in real time (Webhook), triggering the processing of the message in a queue (BullMQ).</li>
  <li className="animation-blur">• **Processing:** The system analyzes the message text using filters to detect inappropriate content. The message is then converted to audio (Text-to-Speech), and the information is updated in the database.</li>
  <li className="animation-blur">• **Organization:** The processed message is sent to a smart queue (Redis), which ensures that no messages are lost and that the chronological order of arrival is maintained.</li>
  <li className="animation-blur">• **Live Display:** A direct connection to the streamer’s screen (WebSocket) visually displays the message in the overlay, plays the generated audio, and, as soon as the display ends, automatically fetches the next message from the queue.</li>
</ul>

### Technical Decisions

<ul className="flex flex-col gap-4">
  <li className="animation-blur">• **Security:** The system stores a detailed and immutable history of financial transactions (Ledger). The balance isn’t just a single number in the database; every transaction is recorded for auditing and security purposes.</li>
  <li className="animation-blur">• **Flexibility:** Widget configurations are stored flexibly (JSON), allowing each widget to be customized differently without requiring major changes to the database for each new option.</li>
  <li className="animation-blur">• **Authentication:** The project uses a hybrid authentication strategy: stateless JWTs containing essential identity and authorization information, such as user ID and roles, while sessions stored in Redis enable active access control. As a result, the system maintains the convenience and scalability of JWT tokens, while still being able to invalidate specific sessions at any time—such as in cases of suspected account compromise—thereby reducing the risk of misuse.</li>
</ul>
